Bootstrap Introduction

Image source: loopple

Bootstrap is an open-source front-end framework originally developed at Twitter and released in 2011. It provides a curated collection of reusable HTML patterns, CSS utilities, and JavaScript components — buttons, navbars, cards, modals, form controls, and a responsive grid system — tested across all major browsers.

Why frameworks accelerate development

Building a responsive, cross-browser UI from scratch requires handling inconsistencies, writing grid logic, crafting accessible components, and testing across devices. Bootstrap encapsulates solutions to these problems into a single well-documented library. A developer applies predefined class names to HTML elements; the framework provides all underlying CSS and JavaScript — reducing time spent on repetitive infrastructure.

CDN setup — including Bootstrap via two lines in HTML
<!-- Bootstrap CSS in <head>, before custom stylesheet -->
<head>
  <link
    rel="stylesheet"
    href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.2/dist/css/bootstrap.min.css"
  />
  <link rel="stylesheet" href="style.css" />
</head>

<!-- Bootstrap JS bundle at end of <body> — includes Popper.js -->
<body>
  <script
    src="https://cdn.jsdelivr.net/npm/bootstrap@5.3.2/dist/js/bootstrap.bundle.min.js"
  ></script>
</body>

Bootstrap Grid System

Bootstrap's grid divides the horizontal space of a .container into 12 equal columns. Content is placed in .col-* elements nested inside .row elements. Each class prefix maps to a responsive breakpoint; below that breakpoint, columns stack vertically.

Class prefix Breakpoint name Minimum width
col- Extra small (default) Any width
col-sm- Small 576 px
col-md- Medium 768 px
col-lg- Large 992 px
col-xl- Extra large 1200 px

Bootstrap Components — Navbar, Cards, and Buttons

Bootstrap components are pre-built, accessible UI patterns assembled by adding specific class names to HTML elements.

Navbar — responsive with collapse toggle
<nav class="navbar navbar-expand-lg navbar-dark bg-primary">
  <div class="container">
    <a class="navbar-brand" href="#">TechJobs Nepal</a>

    <!-- Appears below lg breakpoint; wired to #navContent -->
    <button class="navbar-toggler"
            data-bs-toggle="collapse"
            data-bs-target="#navContent">
      <span class="navbar-toggler-icon"></span>
    </button>

    <div class="collapse navbar-collapse" id="navContent">
      <ul class="navbar-nav ms-auto">
        <li class="nav-item"><a class="nav-link active" href="#">Home</a></li>
        <li class="nav-item"><a class="nav-link" href="#">Jobs</a></li>
        <li class="nav-item"><a class="nav-link" href="#">Apply</a></li>
      </ul>
    </div>
  </div>
</nav>

Job Application Form — Stage 3: Bootstrap Applied

This is Stage 3. The same form from Stages 1 and 2 is now restructured using Bootstrap's grid, form utility classes (.form-label, .form-control, .form-select, mb-3), and the responsive .col-md-6 column system. The custom CSS from Stage 2 has been replaced by Bootstrap classes.

job-application-bootstrap.html — same form rebuilt with Bootstrap 5 classes
<!-- Bootstrap's .container centres content and adds horizontal padding -->
<div class="container py-4">
  <div class="card shadow">

    <!-- .card-header: themed top bar -->
    <div class="card-header bg-primary text-white">
      <h5 class="mb-0">Job Application Form</h5>
      <small>TechJobs Nepal — Stage 3: Bootstrap</small>
    </div>

    <div class="card-body p-4">
      <form novalidate>

        <!-- .row + .col-md-6: two columns on medium+ screens -->
        <div class="row g-3 mb-3">
          <div class="col-md-6">
            <label for="bsName" class="form-label">Full Name *</label>
            <input type="text" class="form-control" id="bsName"
                   placeholder="e.g. Aadarsh K.C." required />
          </div>
          <div class="col-md-6">
            <label for="bsEmail" class="form-label">Email Address *</label>
            <input type="email" class="form-control" id="bsEmail"
                   placeholder="aadarsh@example.com.np" required />
          </div>
        </div>

        <div class="row g-3 mb-3">
          <div class="col-md-6">
            <label for="bsPhone" class="form-label">Phone (optional)</label>
            <input type="tel" class="form-control" id="bsPhone"
                   placeholder="+9779802864772" />
          </div>
          <div class="col-md-6">
            <label for="bsCity" class="form-label">City *</label>
            <input type="text" class="form-control" id="bsCity"
                   placeholder="e.g. Kathmandu"
                   list="bsCityList" required />
            <datalist id="bsCityList">
              <option value="Kathmandu"><option value="Pokhara">
              <option value="Lalitpur"><option value="Nepalgunj">
            </datalist>
          </div>
        </div>

        <div class="row g-3 mb-3">
          <div class="col-md-6">
            <label for="bsPos" class="form-label">Position *</label>
            <!-- .form-select styles a <select> to match Bootstrap inputs -->
            <select class="form-select" id="bsPos" required>
              <option value="">-- Please select --</option>
              <optgroup label="Frontend">
                <option value="frontend">Frontend Developer</option>
                <option value="ui">UI Engineer</option>
              </optgroup>
              <optgroup label="Backend">
                <option value="backend">Backend Developer</option>
                <option value="devops">DevOps Engineer</option>
              </optgroup>
              <optgroup label="Full Stack">
                <option value="fullstack">Full Stack Developer</option>
              </optgroup>
            </select>
          </div>
          <div class="col-md-6">
            <label for="bsExp" class="form-label">Years of Experience *</label>
            <input type="number" class="form-control" id="bsExp"
                   min="0" max="30" placeholder="e.g. 2" required />
          </div>
        </div>

        <div class="mb-3">
          <label for="bsPort" class="form-label">Portfolio URL</label>
          <input type="url" class="form-control" id="bsPort"
                 placeholder="https://github.com/aadarshkc" />
        </div>

        <div class="mb-4">
          <label for="bsMsg" class="form-label">Cover Note *
            <span class="text-muted fw-normal">(min. 20 characters)</span>
          </label>
          <!-- .form-control on textarea gives the same appearance as inputs -->
          <textarea class="form-control" id="bsMsg" rows="4"
                    placeholder="Briefly describe your skills and experience..."
                    required></textarea>
        </div>

        <!-- .d-grid makes the button full-width on all screen sizes -->
        <div class="d-grid">
          <button type="submit" class="btn btn-primary">Submit Application</button>
        </div>

      </form>
    </div>
  </div>
</div>

Compare Stage 3 with Stage 2 (CSS page). The form looks very similar because Bootstrap's default form styles are close to the custom CSS written in Stage 2 — but Stage 3 uses no custom CSS at all. The two-column layout, focus rings, label styles, and full-width button are all provided by Bootstrap's utility classes. In a production project, custom CSS would then be layered on top to apply brand colours.
