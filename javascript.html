What is JavaScript?

Image source: freepnglogos

JavaScript is a high-level, interpreted, dynamically typed programming language governed by the ECMAScript (ES) standard, maintained by ECMA International. It is the only programming language natively supported by all major web browsers, making it the universal language of client-side web development.

The three-layer model

JavaScript completes the three-layer model of the frontend alongside HTML and CSS. HTML defines the structure of a document; CSS defines its appearance; JavaScript defines its behaviour — how elements respond to user interactions, how data is fetched from a server, and how the DOM is updated without a full page reload. This separation of concerns is the foundation of modern web development practice.

Modern JavaScript refers to ECMAScript 2015 (ES6) and later. ES6 introduced significant improvements: block-scoped variable declarations with let and const, arrow functions, template literals, destructuring, the class syntax, and native Promises for asynchronous programming. All examples in this portfolio use ES6+ syntax.

Basic JavaScript Concepts — Variables, Functions, and Events

Variables — const, let, and primitive data types
// const: block-scoped; the binding cannot be reassigned.
// Use for values that should not change after declaration.
const studentName = "Aadarsh K.C.";   // String
const moduleCode   = "CET138";
const creditHours  = 20;               // Number
const isEnrolled   = true;             // Boolean

// let: block-scoped; the binding can be reassigned.
let errorCount = 0;
errorCount = errorCount + 1; // now 1

// undefined: declared but not yet assigned a value.
let grade;
console.log(grade); // undefined

// null: intentionally assigned to indicate "no value".
const result = null;

// typeof: returns the data type as a string.
console.log(typeof studentName); // "string"
console.log(typeof creditHours);  // "number"
console.log(typeof isEnrolled);   // "boolean"

Functions — declaration, arrow function, and conditional logic
// Traditional function declaration — hoisted to the top of its scope,
// meaning it can be called before the line it is defined on.
function greet(name) {
  return `Welcome, ${name}. Module: CET138.`; // template literal
}

// Arrow function (ES6) — concise syntax; no own 'this' binding.
// Suitable for callbacks and short utility functions.
const square = (n) => n * n;

// Multi-line arrow function with conditional logic
const getGrade = (score) => {
  if      (score >= 70) return "Distinction";
  else if (score >= 50) return "Pass";
  else                   return "Fail";
};

console.log(greet("Aadarsh")); // "Welcome, Aadarsh. Module: CET138."
console.log(square(7));         // 49
console.log(getGrade(82));       // "Distinction"
console.log(getGrade(55));       // "Pass"
console.log(getGrade(38));       // "Fail"
Output
Events — addEventListener, click, and input
// addEventListener(eventType, callback) attaches a handler function
// that executes each time the specified event fires on the element.

// 'click' fires once per mouse click or keyboard activation
const btn = document.getElementById('submitBtn');

btn.addEventListener('click', function (event) {
  // event.preventDefault() stops the browser's default action.
  // For a button inside a form, that action is page submission.
  event.preventDefault();
  console.log('Button activated.');
});

// 'input' fires synchronously on every value change (each keystroke)
const nameField = document.getElementById('fullName');

nameField.addEventListener('input', function () {
  console.log('Current value: ' + this.value);
});

Job Application Form — Stage 4: JavaScript Validation

This is Stage 4 — the final stage of the form's progression. The HTML structure from Stage 1, the CSS styling from Stage 2, and the Bootstrap layout from Stage 3 are all retained. JavaScript is now layered on top to intercept the submit event, validate each field, and inject error messages directly into the DOM — all without any page reload.

Below is the complete source code used to power the form validation, followed by the live interactive output.

script.js — client-side validation for the Job Application Form
/* ── Helper functions ────────────────────────────────────── */

function showError(fieldId, message) {
  const field = document.getElementById(fieldId);
  const err   = document.getElementById(fieldId + 'Error');
  // Add red border to the field
  if (field) field.classList.add('field-error');
  // Make the error message visible with text
  if (err)   { err.textContent = message; err.classList.add('show'); }
}

function clearError(fieldId) {
  const field = document.getElementById(fieldId);
  const err   = document.getElementById(fieldId + 'Error');
  if (field) field.classList.remove('field-error');
  if (err)   err.classList.remove('show');
}

/* Regex: checks for @ symbol and at least one dot in the domain */
function isValidEmail(email) {
  return /^[^\s@]+@[^\s@]+\.[^\s@]+$/.test(email);
}

/* Phone: digits, spaces, hyphens, plus, parentheses — 7 to 15 chars */
function isValidPhone(phone) {
  return /^[\d\s\-\+\(\)]{7,15}$/.test(phone);
}

/* ── Attach the submit handler ───────────────────────────── */

const appForm = document.getElementById('jobAppForm');

/* Clear each field's error as the user types — live feedback */
['fullName', 'email', 'phone', 'city', 'position', 'experience', 'portfolio', 'message'].forEach(function(id) {
  const el = document.getElementById(id);
  if (el) {
    el.addEventListener('input',  () => clearError(id));
    el.addEventListener('change', () => clearError(id));
  }
});

appForm.addEventListener('submit', function (event) {

  // Stop the browser navigating to the action URL
  event.preventDefault();

  // Read and trim each field value
  const fullName   = document.getElementById('fullName').value.trim();
  const email      = document.getElementById('email').value.trim();
  const phone      = document.getElementById('phone').value.trim();
  const city       = document.getElementById('city').value.trim();
  const position   = document.getElementById('position').value;
  const experience = document.getElementById('experience').value.trim();
  const message    = document.getElementById('message').value.trim();

  const errorAlert   = document.getElementById('formErrorAlert');
  const successAlert = document.getElementById('formSuccessAlert');

  // Reset previous alert visibility
  errorAlert.classList.remove('show');
  successAlert.classList.remove('show');
  ['fullName', 'email', 'phone', 'city', 'position', 'experience', 'portfolio', 'message'].forEach(clearError);

  let valid = true;

  // Validate each field; accumulate errors rather than stopping early
  if (fullName.length < 2) {
    showError('fullName', 'Full name must be at least 2 characters.');
    valid = false;
  }
  if (!isValidEmail(email)) {
    showError('email', 'Please enter a valid email address.');
    valid = false;
  }
  if (phone && !isValidPhone(phone)) {
    showError('phone', 'Please enter a valid phone number.');
    valid = false;
  }
  if (!city) {
    showError('city', 'Please enter your city.');
    valid = false;
  }
  if (!position) {
    showError('position', 'Please select a position.');
    valid = false;
  }
  if (!experience || isNaN(experience) || Number(experience) < 0) {
    showError('experience', 'Please enter your years of experience.');
    valid = false;
  }
  if (message.length < 20) {
    showError('message', 'Cover note must be at least 20 characters.');
    valid = false;
  }

  if (!valid) {
    // Show summary error banner and scroll it into view
    errorAlert.classList.add('show');
    errorAlert.scrollIntoView({ behavior: 'smooth', block: 'center' });
    return;
  }

  // All fields valid — show success, reset form
  successAlert.classList.add('show');
  appForm.reset();
  successAlert.scrollIntoView({ behavior: 'smooth', block: 'center' });
});

const input   = document.getElementById('dynInput');

const MAX     = 40;

input.addEventListener('input', function () {

Code Explanation — The Validation Script

The table below provides a line-by-line explanation of the key techniques used in script.js to implement client-side validation on the Job Application Form.

Code Explanation
document.addEventListener
('DOMContentLoaded', fn)		 Defers all DOM access until the HTML has been fully parsed. Without this, scripts at the end of <body> could execute before all elements are available.
form.addEventListener
('submit', fn)		 Attaches a callback to the form's submit event. The callback runs whenever the user clicks the submit button or presses Enter inside a field.
event.preventDefault() Cancels the browser's default form-submission behaviour, which would navigate to the action URL and reload the page. This hands full control to JavaScript.
input.value.trim() Reads the field's current text value. .trim() removes leading and trailing whitespace, preventing a field containing only spaces from passing a length check.
/^[^\s@]+@[^\s@]+\.[^\s@]+$/ A regular expression pattern that tests whether a string contains one @ symbol and at least one dot in the domain segment — a lightweight email format check.
el.classList.add('show') Adds the show class to a DOM element. In this portfolio, .show sets display: block, making a hidden error message visible — without modifying the HTML.
el.textContent = msg Sets the visible text of an element. Using textContent rather than innerHTML is a security practice — it prevents any injected content from being interpreted as HTML, avoiding cross-site scripting vulnerabilities.
el.scrollIntoView
({ behavior: 'smooth' })		 Scrolls the viewport smoothly to bring the error alert into view. On mobile devices where the form may extend below the fold, this ensures the user sees the error message.
form.reset() Restores all form controls to their default state. Called after a successful submission to prepare the form for potential re-use without a page reload.
Client-side validation as implemented here improves the user experience by providing immediate feedback, but it is not a security measure. Any data submitted from a browser can be intercepted and modified. Production applications must always re-validate inputs on the server before processing or persisting them.
