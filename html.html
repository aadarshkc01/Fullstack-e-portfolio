What is HTML?

Image source: toppng

HyperText Markup Language (HTML) is the standard language used to define the structure and content of web pages. It organises content into a hierarchy of elements, each identified by an opening tag such as <h1> and a corresponding closing tag </h1>. Void elements such as <img> and <input> are self-closing and carry no content between tags.

HTML is not a programming language

HTML contains no conditional logic, loops, or computational operations. Its sole purpose is to construct the Document Object Model (DOM) — a hierarchical tree of nodes that the browser renders as a visual page. CSS subsequently applies style rules to those nodes, and JavaScript reads and manipulates them at runtime.

HTML5, maintained as a living standard by the WHATWG, introduced a set of semantic elements — including <header>, <nav>, <main>, <article>, <section>, and <footer> — that communicate the meaning of content, not merely its appearance. Semantic markup improves accessibility for assistive technologies, aids search-engine indexing, and makes codebases significantly easier to maintain.

Basic HTML Structure

Every valid HTML5 document requires four components. Each has a distinct, mandatory purpose:

boilerplate.html 
<!-- DOCTYPE: not an HTML tag — a processing instruction.
     Tells the browser to render using HTML5 standards mode.
     Without it, browsers may fall back to "quirks mode". -->
<!DOCTYPE html>

<!-- lang="en" identifies the page language for screen readers. -->
<html lang="en">

<head>
  <meta charset="UTF-8" />
  <meta name="viewport" content="width=device-width, initial-scale=1.0" />
  <title>My Page</title>
  <link rel="stylesheet" href="style.css" />
</head>

<body>
  <h1>Hello, World!</h1>
  <script src="script.js"></script>
</body>
</html>

Important HTML Tags

Each example below shows the HTML markup followed directly by how a browser renders it using only the built-in default stylesheet — no custom CSS applied.

Tag Category Purpose Key attributes
<h1>–<h6> Block Heading hierarchy. One <h1> per page; screen readers and search engines use this hierarchy. Global: id, class, lang
<p> Block Paragraph of body text. Browser adds default margin between paragraphs. id, class, style
<strong>, <em> Inline <strong> = strong importance (bold). <em> = stress emphasis (italic). Both are semantic; distinct from <b> and <i>. id, class
<fieldset> Block Groups related form controls visually and semantically. Renders a border around the group. Used with <legend>. name, disabled (disables all child controls)
<legend> Block Caption for a <fieldset> group. Screen readers announce it before each control inside the fieldset. id, class
<datalist> Block Provides a predefined list of suggestions for an <input>. The user can still type freely; the list is advisory only. id (must match list on the input)
<optgroup> Inline Groups <option> elements inside a <select> with a labelled separator. Improves scanability of long drop-downs. label (group heading text), disabled
<a> Inline Hyperlink to a URL, email, tel, or anchor. Without href acts as a placeholder. href, target, rel, download
<img> Void Embeds an image. The alt attribute is mandatory for accessibility and as fallback text. src, alt (required), width, height, loading="lazy"
<table>, <thead>, <tbody> Block Represents tabular data. Must not be used for page layout. <thead> groups header rows; <tbody> groups data rows. id, class; <th scope="col|row">
<form> Block Container for interactive controls. action and method determine where and how data is sent. action, method, novalidate, enctype
<input> Void Versatile void element that renders different controls based on type. The most common element on the web. type, id, name, placeholder, required, pattern, list, aria-describedby
<select> Inline Drop-down menu of <option> elements. Can be grouped with <optgroup>. id, name, required, multiple
<textarea> Inline Multi-line text input for longer free-form content. Has an opening and closing tag unlike <input>. rows, cols, placeholder, required, maxlength
<button> Inline Clickable button. type="submit" sends the form; type="button" requires a JS listener. type, disabled, form

     Screen readers announce the legend before each child control. -->

Job Application Form — Stage 1: Raw HTML

This is Stage 1 of a four-stage progression. The same form will be enhanced with CSS styling (Stage 2), Bootstrap layout (Stage 3), and JavaScript validation (Stage 4) in the sections that follow. Here it is rendered with the browser's default stylesheet only — demonstrating that HTML defines structure and meaning, not appearance.

This version uses <fieldset> and <legend> to group related fields, <datalist> for city suggestions, <optgroup> to group roles, and aria-describedby / aria-required for full accessibility.

job-application-form.html — plain HTML, no CSS applied
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8" />
  <meta name="viewport" content="width=device-width, initial-scale=1.0" />
  <title>Job Application — TechJobs Nepal</title>
</head>
<body>

  <h1>Job Application Form</h1>
  <p>TechJobs Nepal — Apply for a developer position below.</p>
  <p><small>Fields marked * are required.</small></p>

  <form action="/submit-application" method="post" novalidate>

    <!-- fieldset groups personal information fields semantically -->
    <fieldset>
      <legend>Personal Information</legend>

      <label for="fullName">Full Name *</label><br />
      <input type="text" id="fullName" name="fullName"
             placeholder="e.g. Aadarsh K.C."
             required aria-required="true"
             aria-describedby="fullNameError" /><br /><br />

      <label for="email">Email Address *</label><br />
      <input type="email" id="email" name="email"
             placeholder="aadarsh@example.com.np"
             required aria-required="true"
             aria-describedby="emailError" /><br /><br />

      <!-- type="tel" shows numeric keypad on mobile devices -->
      <label for="phone">Phone Number (optional)</label><br />
      <input type="tel" id="phone" name="phone"
             placeholder="+9779802864772"
             pattern="[+\d\s\-()]{7,15}" /><br /><br />

      <!-- datalist provides Nepali city suggestions -->
      <label for="city">City *</label><br />
      <input type="text" id="city" name="city"
             placeholder="e.g. Kathmandu"
             list="cityList" required />
      <datalist id="cityList">
        <option value="Kathmandu"><option value="Pokhara">
        <option value="Lalitpur"><option value="Nepalgunj">
        <option value="Bharatpur">
      </datalist><br /><br />
    </fieldset>

    <br />

    <!-- Second fieldset groups application-specific fields -->
    <fieldset>
      <legend>Application Details</legend>

      <!-- optgroup groups job roles by category -->
      <label for="position">Position Applied For *</label><br />
      <select id="position" name="position" required>
        <option value="">-- Please select --</option>
        <optgroup label="Frontend">
          <option value="frontend">Frontend Developer</option>
          <option value="ui">UI Engineer</option>
        </optgroup>
        <optgroup label="Backend">
          <option value="backend">Backend Developer</option>
          <option value="devops">DevOps Engineer</option>
        </optgroup>
        <optgroup label="Full Stack">
          <option value="fullstack">Full Stack Developer</option>
        </optgroup>
      </select><br /><br />

      <label for="experience">Years of Experience *</label><br />
      <input type="number" id="experience" name="experience"
             min="0" max="30" placeholder="e.g. 2" required /><br /><br />

      <label for="portfolio">Portfolio URL</label><br />
      <input type="url" id="portfolio" name="portfolio"
             placeholder="https://github.com/aadarshkc" /><br /><br />

      <label for="message">Cover Note * (min. 20 characters)</label><br />
      <textarea id="message" name="message"
                rows="5" cols="45"
                placeholder="Briefly describe your skills and experience..."
                required minlength="20"></textarea><br /><br />
    </fieldset>

    <br />
    <button type="submit">Submit Application</button>

  </form>
</body>
</html>
