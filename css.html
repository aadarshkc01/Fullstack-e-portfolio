CSS — Cascading Style Sheets

Types of CSS, core properties, the Box Model, Flexbox, media queries, and the styled form.

1

What is CSS?

Image source: animalia life

Cascading Style Sheets (CSS) is a declarative language that describes the visual presentation of HTML documents. It controls colour, typography, spacing, layout, and animation — all independently of the document's semantic structure. This separation of concerns is a foundational principle of web standards: HTML defines what content is; CSS defines how it looks.

The "cascading" mechanism

When multiple CSS rules target the same element, the browser resolves the conflict using three criteria: origin (author, user, or browser stylesheet), specificity (how precisely the selector targets an element — ID selectors outrank class selectors), and source order (later rules override earlier ones at equal specificity).

Types of CSS

CSS can be attached to an HTML document in three ways. Each method has a different scope and appropriate context.

Inline CSS — style attribute on a single element
<!-- Highest specificity; appropriate for one-off overrides only. -->
<p style="color: #594CE0; font-weight: 600; font-size: 1.05rem;">
  This paragraph is styled inline.
</p>

<button style="background:#594CE0; color:white;
         padding:8px 18px; border:none; border-radius:4px;">
  Submit
</button>
Output

This paragraph is styled inline.

Internal CSS — style element in the head
<!-- Rules apply to all matching elements on this page only.
     Useful for single-page prototypes; does not scale. -->
<head>
  <style>
    body { font-family: Arial, sans-serif; background: #f0f4f8; }
    h2   { color: #594CE0; border-bottom: 2px solid #594CE0; }
    .note { background: #eeedfb; border-left: 4px solid #594CE0; }
  </style>
</head>
Output

External CSS — linked stylesheet (recommended)
<!-- All pages sharing this file receive the same styles.
     Browser caches it after first download — better performance. -->
<head>
  <link rel="stylesheet" href="style.css" />
</head>

  background: #F8F9FA;

CSS Layout — Flexbox and Media Queries

Flexbox

Flexbox is a one-dimensional layout algorithm applied to a container with display: flex. Child elements are arranged along a main axis with precise control over alignment and spacing.

Flexbox — navbar: brand left, links right
.navbar {
  display: flex;
  justify-content: space-between; /* push children apart */
  align-items: center;            /* vertically centre */
  background: #594CE0;
  padding: 0.75rem 1.5rem;
}

.navbar-links {
  display: flex;
  gap: 1.25rem; /* consistent spacing between links */
}

Responsive Design with Media Queries

A media query conditionally applies CSS rules based on the viewport width. The mobile-first approach writes base styles for narrow screens and uses min-width queries to progressively enhance for wider devices.

@media queries — responsive breakpoints for card grid
/* ─── Mobile-first base: single column on small screens ─── */
.card-grid {
  display: grid;
  grid-template-columns: 1fr; /* 1 column default */
  gap: 1rem;
}

/* ─── Small (≥576px): two columns — tablets in portrait ─── */
@media (min-width: 576px) {
  .card-grid { grid-template-columns: 1fr 1fr; }
}

/* ─── Large (≥992px): three columns — desktop / laptop ─── */
@media (min-width: 992px) {
  .card-grid { grid-template-columns: 1fr 1fr 1fr; }
}

/* ─── Form row: two columns → stacks on mobile ─── */
.form-row {
  display: flex;
  gap: 1rem;
}
.form-row .form-group { flex: 1; }

@media (max-width: 768px) {
  .form-row { flex-direction: column; }

  /* Reduce font sizes on very small screens */
  h1 { font-size: 1.8rem; }
  .site-header { padding: 0 1rem; }
}

@media (max-width: 480px) {
  /* Stacked single column on phones */
  .card-grid { grid-template-columns: 1fr; }
  .hero      { padding: 5rem 1.25rem 3rem; }
}
The mobile-first approach is the industry standard: write styles for the narrowest viewport first, then progressively add layout with min-width queries. This ensures the smallest devices always receive a usable experience without downloading unnecessary overrides.

/* ── Custom properties (design tokens) ─────────────────── */

/* ── Form groups ─────────────────────────────────────────── */

  margin-bottom: 0.32rem;

.form-group input,
.form-group select,

  .form-row { flex-direction: column; }
}

/* ── Submit button ───────────────────────────────────────── */

  padding: 0.72rem;
  background: var(--navy);
  color: #fff;
  border: none;
